Question: 1. Discuss the scope of var, let, and const


Answer:

Var: The var statement declares function-scoped or globally-scoped variables. It's allow to reassign another value of a variable. It's can be declared inside a { } block and can be accessed from outside the block. It's can be updated and re-declared.

Let: Variables declared as let are in the block scope. It's allow to reassign another value and reset the new value of a variable. It's can be changed or declared again. It's can be updated but not re-declared.

Const: The scope of const declarations is blocked. It is impossible to modify or redeclare. It's value will not change. It's cannot be changed or declared again.

Question: 2. Tell us the use cases of null and undefined


Answer:

Null: Null is an object in JavaScript. The null value represents the intentional absence of any object value. A null value in a relational database is used when the value in a column is unknown or missing.

Undefined: Undefined is one of JavaScript's primitive types. A function parameter that is not passed will be undefined. When a variable that is not initialized or not assigned with any value will give undefined. A function returns undefined if a value was not returned. Variable name is declared but not assigned with any value, so it gets an Undefined value.

Question: 3. What do you mean by REST API?


Answer:

Rest API: REST stand for Representation State Transfer and API stands for Application programming Interface. Rest API is an interface that two computer systems use to exchange information securely over the internet. It is basically a way of building web API's in logical way making them easy to consume because REST uses less bandwidth, making it more suitable for efficient internet usage. A REST API is an application program interface that use HTTP requests to GET,PUT and DELETE data. A REST API uses commands to obtain resources. The state of a resource at any given timestamp is called a resource representation. REST APIs provide simple, uniform interfaces because they can be used to make data, content, algorithms, media, and other digital resources available through web URLs.
